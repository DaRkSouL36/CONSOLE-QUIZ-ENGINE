## 📘 PROJECT OVERVIEW

THIS REPOSITORY SERVES AS A MASTERCLASS IN C++ SOFTWARE ENGINEERING. INSTEAD OF RELYING ON PRE-BUILT FRAMEWORKS, THIS PROJECT MANUALLY IMPLEMENTS GAME LOOPS, DYNAMIC STRING PARSING, INPUT SANITIZATION, AND HIGH-PRECISION TIMERS. IT IS DESIGNED TO PROVE THAT ROBUST, CONFIGURABLE, AND ENGAGING APPLICATIONS CAN BE BUILT USING STRICTLY THE STANDARD TEMPLATE LIBRARY (STL).

---

## ✨ KEY FEATURES

- **ZERO EXTERNAL LIBRARIES:** RELIES STRICTLY ON THE C++ STANDARD LIBRARY (`<iostream>`, `<fstream>`, `<vector>`, `<chrono>`, `<algorithm>`).
- **DYNAMIC CONFIGURATION PIPELINE:** INTERACTIVE MENUS TO FILTER QUESTIONS BY SUBJECT CATEGORY, DIFFICULTY LEVEL, AND SESSION LIMITS.
- **ASYNCHRONOUS-STYLE TIMERS:** UTILIZES `<chrono>` TO TRACK USER RESPONSE TIMES DOWN TO THE MILLISECOND, AUTOMATICALLY FAILING SLOW ANSWERS WITHOUT BLOCKING THE MAIN THREAD.
- **50/50 LIFELINE SYSTEM:** A STRATEGIC HINT MECHANIC THAT DYNAMICALLY MUTATES THE OBJECT STATE TO ELIMINATE TWO INCORRECT OPTIONS WHILE THE TIMER CONTINUES TO RUN.
- **PERSISTENT LEADERBOARDS:** AUTOMATIC FILE I/O TO LOG SCORES, CALCULATE ACCURACY PERCENTAGES, RECORD TIMESTAMPS, AND SORT PLAYERS USING CUSTOM LAMBDA FUNCTIONS.
- **POST-GAME ANALYTICS:** OFFERS PLAYERS A DETAILED REVIEW OF INCORRECT ANSWERS, COMPARING THEIR INPUT AGAINST THE CORRECT DATA AND SHOWING TIMEOUT METRICS.

## 🧠 CORE MECHANICS IMPLEMENTED

### 1. DYNAMIC FILE PARSING (`std::ifstream`)
- IMPLEMENTED A ROBUST 8-LINE BLOCK PARSER TO EXTRACT CATEGORY, DIFFICULTY, TEXT, AND OPTIONS WHILE SAFELY HANDLING BLANK LINES AND EOF MARKERS.

### 2. HARDWARE-LEVEL RANDOMIZATION (`std::mt19937`)
- USES `std::random_device` TO SEED THE MERSENNE TWISTER ENGINE, ENSURING CRYPTOGRAPHICALLY UNPREDICTABLE QUESTION SHUFFLING ON EVERY PLAYTHROUGH.

### 3. PRECISION TIMING (`std::chrono`)
- UTILIZES `std::chrono::steady_clock` TO MEASURE THE EXACT `duration<double>` BETWEEN RENDER AND INPUT, ENFORCING DYNAMIC TIMEOUTS BASED ON SELECTED DIFFICULTY (10S / 20S / 30S).

### 4. ADVANCED SORTING ALGORITHMS (`std::sort`)
- EMPLOYS CUSTOM LAMBDA FUNCTIONS TO SORT THE LEADERBOARD VECTOR DESCENDING BY WIN PERCENTAGE, USING RAW SCORE AND ATTEMPT COUNTS AS TIE-BREAKERS.

## ⚙️ HOW IT WORKS



1. **INITIALIZATION:** `main.cpp` INVOKES `FileLoader` TO INGEST AND PARSE `Questions.txt` INTO A `std::vector<Question>`.
2. **MENU PIPELINE:** THE SYSTEM EXTRACTS UNIQUE CATEGORIES USING `std::set`, ALLOWING THE USER TO FILTER AND SLICE THE DATASET BEFORE INITIATING THE ENGINE.
3. **THE GAME LOOP:** `QuizManager` TAKES CONTROL. IT SHUFFLES THE FILTERED POOL, RENDERS UI, STARTS THE CHRONO CLOCK, AND WAITS FOR INPUT (OR A LIFELINE TRIGGER).
4. **EVALUATION & LOGGING:** ANSWERS ARE COMPARED, TIMERS ARE CHECKED, AND DETAILED ATTEMPT HISTORIES ARE LOGGED INTO A PRIVATE `AnswerRecord` STRUCT.
5. **TERMINATION:** PERCENTAGES ARE CALCULATED, INCORRECT ANSWERS ARE REVIEWED, AND THE USER'S SCORE IS APPENDED TO `HighScores.txt` WITH A LOCAL SYSTEM TIMESTAMP.
